Partie I — Introduction au Web Sémantique

Le Web Sémantique : contexte et enjeux

Le Web Sémantique est une extension du Web actuel, proposée en 2001 par Sir Tim Berners-Lee, James Hendler et Ora Lassila dans l'article fondateur publié dans Scientific American. L'idée directrice est de rendre le contenu du Web non plus seulement lisible par des humains, mais interprétable par des machines. Là où le Web traditionnel publie des documents HTML destinés à l'affichage, le Web Sémantique publie des données structurées dotées d'une signification formelle, exploitables par des agents logiciels autonomes.

Le problème fondamental que cherche à résoudre le Web Sémantique est celui de l'ambiguïté. Une même ressource peut être décrite de multiples façons sur différents sites ; un moteur de recherche ne sait pas si « Chanel » désigne la styliste française, la chaîne de télévision ou un prénom. Grâce à des identifiants universels (URI/IRI), des vocabulaires partagés (ontologies) et des langages de représentation formels, le Web Sémantique permet aux machines de relier, croiser et raisonner sur des données hétérogènes.

Définition — Ontologie (informatique)
Une ontologie est une spécification formelle et explicite d'une conceptualisation partagée d'un domaine de connaissance (Gruber, 1993). Elle définit les concepts, leurs relations, leurs propriétés et les contraintes qui régissent leur utilisation.

L'architecture du Web Sémantique est souvent représentée par la Layer Cake de Berners-Lee, une pile de couches technologiques superposées allant des URI et Unicode en base jusqu'aux interfaces de confiance en sommet, en passant par RDF, OWL et les règles logiques. Ce projet s'appuie sur les couches RDF, OWL et SPARQL.

RDF — Resource Description Framework

RDF (Resource Description Framework) est le modèle de données fondamental du Web Sémantique, standardisé par le W3C. Il représente l'information sous forme de triplets de la forme sujet – prédicat – objet, qui constituent ensemble un graphe orienté étiqueté.

ComposantRôleExemple
SujetLa ressource décrite:Gabrielle_Chasnel
PrédicatLa propriété ou relation:paysOrigine
ObjetLa valeur ou ressource liée"France"
 Table 1 — Structure d'un triplet RDF

Chaque ressource est identifiée par un IRI (Internationalized Resource Identifier), garantissant l'unicité mondiale de l'identifiant. RDF dispose de plusieurs syntaxes de sérialisation : Turtle (lisible par l'humain), N-Triples, JSON-LD, et RDF/XML — le format utilisé dans ce projet. 

# Exemple de triplets RDF en syntaxe Turtle
@prefix mode: <http://www.semanticweb.org/manel/ontologies/2025/1/la_mode#> .
@prefix xsd:  <http://www.w3.org/2001/XMLSchema#> .

mode:Gabrielle_Chasnel
    a                   mode:Créateur_de_Mode ;
    mode:nom            "Coco Chanel" ;
    mode:paysOrigine    "France" ;
    mode:dateNaissance  "1883-08-19T00:00:00"^^xsd:dateTime .

RDFS (RDF Schema) enrichit RDF en ajoutant des primitives de hiérarchie : rdfs:subClassOf pour les classes, rdfs:subPropertyOf pour les propriétés, rdfs:domain et rdfs:range pour contraindre les propriétés. Toutefois, RDFS ne permet pas d'exprimer des contraintes de cardinalité, des disjonctions ou des équivalences — d'où la nécessité d'OWL.

OWL — Web Ontology Language

OWL (Web Ontology Language) est le standard W3C pour la représentation d'ontologies riches sur le Web. Basé sur les Logiques de Description (Description Logics), il offre une expressivité formelle permettant le raisonnement automatique : inférence de nouvelles connaissances, détection de contradictions, classification de concepts.

OWL 2 — Profils et expressivité

OWL 2 définit plusieurs profils selon le niveau d'expressivité requis : OWL 2 DL (le plus complet, utilisé dans ce projet), OWL 2 EL, OWL 2 QL et OWL 2 RL. Ce projet utilise OWL 2 DL, qui offre un équilibre entre expressivité et décidabilité.

Raisonnement automatique

Un des grands atouts d'OWL est la possibilité de connecter un raisonneur (reasoner) à l'ontologie. Les raisonneurs OWL 2 DL les plus utilisés sont HermiT (intégré à Protégé) et Pellet. Le raisonneur peut :

  • Classifier automatiquement les individus dans leurs classes les plus spécifiques ;
  • Détecter les incohérences (classes insatisfiables, violations de disjonctions) ;
  • Inférer de nouvelles propriétés à partir des axiomes déclarés ;
  • Vérifier la consistance globale de l'ontologie.

Dans ce projet, le raisonneur HermiT détecte automatiquement que Coco_Chanel et Gabrielle_Chasnel désignent la même personne grâce à l'axiome owl:sameAs.

SPARQL — Langage de requête

SPARQL (SPARQL Protocol and RDF Query Language) est le standard W3C pour interroger des graphes RDF, à la manière de SQL pour les bases de données relationnelles. SPARQL 1.1 (2013) permet les requêtes SELECT, CONSTRUCT, ASK et DESCRIBE, ainsi que les mises à jour via SPARQL Update.

Une requête SPARQL repose sur des patterns de triplets contenant des variables (préfixées par ?) qui sont liées aux données du graphe lors de l'exécution. 

# Requête : Créateurs de Mode avec nom, date, pays
PREFIX mode: <http://www.semanticweb.org/manel/ontologies/2025/1/la_mode#>
PREFIX xsd:  <http://www.w3.org/2001/XMLSchema#>

SELECT ?nom ?dateNaissance ?paysOrigine
WHERE {
  ?createur  a                   mode:Créateur_de_Mode .
  ?createur  mode:nom            ?nom .
  ?createur  mode:dateNaissance  ?dateNaissance .
  ?createur  mode:paysOrigine    ?paysOrigine .
}
ORDER BY ?dateNaissance

OPTIONAL { }Correspondance optionnelle (JOIN gauche)

Partie II — Ontologie de la Mode

Présentation du domaine et motivations

Choix du sujet

Le domaine de la mode vestimentaire a été choisi précisément parce qu'il constitue l'un des sujets les moins représentés dans la littérature des ontologies du Web Sémantique. Une revue de la littérature montre que la quasi-totalité des ontologies de domaine publiées concerne des secteurs comme la médecine (SNOMED CT, Gene Ontology), le droit, la géographie ou les sciences formelles. La mode, en revanche, malgré son importance économique mondiale (estimée à plus de 1 500 milliards de dollars) et son rôle sociologique majeur, n'a fait l'objet que de très rares formalisations ontologiques.

Cette lacune constitue précisément la valeur ajoutée de ce projet : proposer une ontologie structurée, extensible et interrogeable pour un domaine riche en concepts, en relations et en dimensions sociologiques.

Périmètre et objectifs

L'ontologie modélise trois grands axes du domaine de la mode :

  • L'industrie : les créateurs de mode, les marques et les tendances qui structurent le marché ;
  • La sociologie : les mécanismes d'adoption, de diffusion et de signification sociale de la mode ;
  • Les individus concrets : des personnalités réelles (Coco Chanel, Virgil Abloh, Yves Saint Laurent) et des tendances identifiées (Denim universel, Mode des Sneakers).

Conception de l'ontologie

Méthodologie

La conception a suivi une approche top-down : définition des concepts les plus généraux en premier, puis raffinement progressif vers les concepts spécifiques. Les étapes ont été :

  1. Identification du domaine : analyse des concepts clés de la mode (créateur, marque, tendance, style) ;
  2. Collecte des ressources : dictionnaires de la mode, articles de sociologie de la mode (Simmel, Lipovetsky) ;
  3. Modélisation des classes : deux racines (industrie et sociologie), avec sous-hiérarchies ;
  4. Définition des propriétés : propriétés objet (relations) et propriétés de données (attributs) ;
  5. Instanciation : création des individus nommés avec leurs propriétés ;
  6. Axiomatisation : ajout des contraintes de disjonction, de fonctionnalité, et de l'identité owl:sameAs.

IRI de base

L'ontologie est identifiée par l'IRI de base : 

http://www.semanticweb.org/manel/ontologies/2025/1/la_mode

Toutes les entités de l'ontologie sont référencées par des IRI construits à partir de cette base, en ajoutant un fragment identifiant (#Créateur_de_Mode, #nom, etc.).

Héritage multiple

Les classes Nom, Description et PaysOrigine illustrent l'héritage multiple autorisé par OWL. Par exemple, Nom est sous-classe simultanément de Créateur_de_Mode, Marque, Tendance, Affirmation, Autonomie, Impérialisation, Personnalisation et Symbolisation, car le concept de « nom » est transversal à toutes ces entités.

owl:sameAs — Identité entre individus

L'axiome owl:sameAs entre Coco_Chanel et Gabrielle_Chasnel illustre le principe de l'hypothèse du monde ouvert (OWA) propre à OWL. Contrairement aux bases de données classiques (hypothèse du monde fermé), OWL suppose qu'une même entité peut être désignée par plusieurs URIs différents. L'axiome owl:sameAs permet au raisonneur de fusionner les informations des deux individus et d'inférer l'ensemble de leurs propriétés communes. 

<!-- Déclaration d'identité entre deux individus -->
<owl:NamedIndividual rdf:about="...#Coco_Chanel">
  <rdf:type    rdf:resource="...#Nom"/>
  <owl:sameAs rdf:resource="...#Gabrielle_Chasnel"/>
</owl:NamedIndividual>

Requête 1 — Hiérarchie des classes

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?sousClasse ?superClasse
WHERE {
  ?sousClasse rdfs:subClassOf ?superClasse .
  FILTER (!isBlank(?sousClasse) && !isBlank(?superClasse))
}
ORDER BY ?superClasse

Discussion et perspectives

Résultats obtenus

L'ontologie développée répond aux objectifs initiaux : elle offre une représentation formelle du domaine de la mode, exploitable par des machines et interrogeable via SPARQL. Le raisonneur HermiT valide la cohérence de l'ontologie et détecte automatiquement l'identité entre Coco_Chanel et Gabrielle_Chasnel.

Limites

Les principales limites de cette version sont : (1) le nombre restreint d'individus, qui pourrait être étendu à des dizaines ou centaines de créateurs, marques et tendances ; (2) l'absence de relations objet instanciées entre individus (seules les propriétés de données sont peuplées) ; (3) l'absence d'alignement avec des ontologies existantes telles que FOAF, DBpedia ou Wikidata.

Perspectives d'extension

  • Enrichissement des données : ajout de dizaines de créateurs, marques historiques et tendances contemporaines ;
  • Alignement ontologique : liens owl:sameAs vers DBpedia/Wikidata pour chaque créateur ;
  • Règles SWRL : règles inférant automatiquement le style d'une tendance à partir de ses attributs ;
  • Endpoint SPARQL : déploiement sur Apache Jena Fuseki pour interrogation distante ;
  • Intégration à un système de recommandation mode basé sur le raisonnement ontologique.

Conclusion

Ce projet a permis de concevoir et d'implémenter une ontologie formelle du domaine de la mode vestimentaire en utilisant les technologies du Web Sémantique — OWL 2, RDF/XML et SPARQL 1.1 — avec l'éditeur Protégé 5. L'ontologie modélise les entités industrielles (créateurs, marques, tendances) et les phénomènes sociologiques (affirmation, autonomie, impérialisation, personnalisation, symbolisation), et les relie par un réseau de propriétés objet et de données.

La valeur de ce travail réside dans l'originalité du domaine choisi — la mode, très peu représentée dans la littérature ontologique — ainsi que dans la rigueur de la modélisation : hiérarchies de classes bien définies, contraintes de disjonction, propriétés fonctionnelles, et démonstration de l'identité via owl:sameAs.

Ce projet constitue une première pierre vers une base de connaissances sémantique complète sur la mode, ouvrant des perspectives dans les domaines de la recommandation, de la recherche d'information et de l'analyse sociologique computationnelle.
